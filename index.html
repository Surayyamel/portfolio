Workout Log

Tech stack

About the project

Keep track of your workouts, sets, reps and weights by saving them in this calendar based CRUD application!
I aimed to create an alternative to a paper-based notebook or notes app, specifically tailored to weights-tracking. I used the React library and Express framework alone with the PostgreSQL database. I used a Google OAuth log-in strategy to allow users to easily log-in with their Google account.

How it works

Log-in, select a date and complete the form that appears next to it. Enter a workout name for each day, and as many exercises as needed. After adding the exercise name and the number of sets, subsequent fields will appear dynamically to allow the user to add reps/weights relative to the number of sets entered.
The date appears green if it contains any data, to keep track of which days have workouts saved.

Difficulties encountered Solutions
Security Used express-rate-limit package in authentication route to limit requests. Used validator.js package for user input validation and sanitization. Disabled X-Powered-By header.
A dynamic form that relies on user input Opted for using npm package 'react-hook-form', simplifying the dynamic aspect.
Storing and retrieving the data (reps, weights) in the correct matching order to be displayed in a table format. Assigned a specific index to each weight and rep names and ensured that the data gets inserted into DB consistently in the correct order, so that it could be retrieved in order.
Designing the database to ensure a link between the relevant users, dates, sets, reps and weights and limit database calls. Used Primary and Forein keys to create links between relevant data, and used table unions to retrieve data.
Hosting on Heroku: OAuth log-in and database pooling not working . Was not able to freely exchange cookies between seperately hosted front-end and back-end. Added SameSite=None and Secure settings to cookie at creation. Initially used pooling to handle database requests, but this was not supported by Heroku. Changed to connect and end connection on each CRUD route.

Thoughts and improvements

While this project is functioning on all other browsers, it has been marked as deceptive by Google Chrome, so I am delving deeper into App Security to figure out the reason, and to find any security holes in my project. (I have currently taken the project offline)
After asking for user feedback, I realised that some functionalities that made sense to me, were confusing to others. I am thinking of adding more user guidence. The seperate 'workout name' and 'exercise' form could lead to confusion, as they appear simultaneously.
I would like to add another log-in option for users that may not want to use a Google account. I will need to find a way to integrate these users into the database and edit the database accordingly. I would also need to store and encrypt user passwords.
I learned a lot while making this project as it is my first full-stack project. Some things I learned are:

  • Plan thoroughly - spending more time at the beginning will save me time in the end
  • "This works" - make sure it is a solution that will hold up and is not just a temporary fix
  • Establish design patterns before starting to write to ensure consistency (React Components, CSS classes, naming variables etc)
  • Keep notes of errors and how to fix them for future reference

Currency Converter

Tech stack

About the project

Enter a numeric value and pick currencies to convert!
This project uses the ExchangeRate-API for exchange rate data. Originally a front-end React app, I later created an Express server for security as the API could no longer be used without an API key.
The request is sent to the server with the 'to' and 'from' country codes. The server then sends a request for the exchange rate, which is sent back to the front-end. The currency exchange is calculated on the front-end using the requested exchange rate.

Weather App

Tech stack

About the project

Enter a location to search for the current weather!
This Node.js app uses Express and Handlebars to create the views.
There are two API calls - the Mapbox API does the geocoding, converting location text into geographic coordinates, and the Weatherstack API uses latitudes and longitudes to get the weather data.
Depending on the weather result keywords, a different animated SVG image will appear (sunny, cloudy or rainy).
