About the project

Keep track of your workouts, sets, reps and weights by saving them in this calendar based CRUD application!
I aimed to create an alternatve to a paper-based notebook or notes app, specifically tailored to weights-tracking. I used the React library and Express framework with PostgreSQL as a database, opting for a structred approach. I used a Google OAuth log-in strategy to allow users to easily log-in with their Google account.

How it works

Log-in, select a date and complete the form that appears next to it. Enter a workout name for each day, and as many exercises as needed. After adding the exercise name and the number of sets, subsequent fields will appear dynamically to allow the user to add reps/weights relative to the number of sets entered.
The date appears green if it contains any data, to keep track of which days have workouts saved.

Difficulties encountered Solutions
A dynamic form dependant on user input Opted for using npm package 'react-hook-form', simplifying the dynamic aspect.
Storing and retrieving the data (reps, weights) in the correct matching order Assigned a specific index to each weight and rep names and ensured that the data gets inserted into DB in the correct order, so that it could be retrieved in order.
Designing the database to ensure a link between the relevant users, dates, sets, reps and weights Linked reps and weights to exercise using an exercise ID. Linked exercises and workout names to users using a user ID. Exercise and workout names also keep track of their selected date.
Hosting on Heroku: OAuth log-in not working on hosted site Was not able to freely exchange cookies between seperately hosted front-end and back-end. Added SameSite=None and Secure settings to cookie at creation.
Hosting on Heroku: Database pooling not supported Initially used pooling to handle database requests, but this was not supported by Heroku. Changed to connect and end connection on each CRUD route.
